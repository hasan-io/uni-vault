
# 🎓 Centralized Academic & Co-Curricular Record System

A unified, role-based web platform designed for universities to **manage, verify, and track student academic and extracurricular records** in a single, centralized system.

---

## 🚀 Problem Statement

Higher Education Institutions often struggle with **fragmented student data** — academic records, achievements, internships, and extracurricular activities are scattered across departments, emails, and physical files.

This leads to:

* Inefficient tracking of student progress
* Difficulty in verifying achievements
* Lack of a unified student profile
* Time-consuming report generation

---

## 💡 Solution

This project provides a **centralized digital platform** where:

* Students can manage and track their achievements
* Faculty can verify records and manage academics
* Admins can oversee and analyze institution-wide data

All data is **secure, verified, and synchronized in real-time**.

---

## 🧑‍💻 User Roles & Features

### 👨‍🎓 Student

* View dashboard with academic & activity insights
* Add achievements with proof (certificates)
* Track verification status (Pending / Approved / Rejected)
* View academic records (marks, grades, attendance)
* Generate downloadable portfolio (PDF)
* Receive notifications (verification updates, marks added)
* AI Resume Builder (suggestions based on profile data)
* AI Skill Assistant (guidance for upskilling)
* Leaderboard & engagement tracking

---

### 👨‍🏫 Faculty

* Verify student achievements
* Add academic records (marks, grades)
* Mark attendance per subject
* View student profiles (academic + extracurricular)
* Filter students by course, semester, and section
* Bulk upload marks via Excel

---

### 🧑‍💼 Admin

* Manage students and faculty accounts
* Assign roles and map subjects
* Define academic structure (departments, courses, sections)
* Generate credentials for faculty
* Access full institutional analytics
* Monitor engagement and performance trends

---

## 🔐 Authentication & Access Control

* Role-based login system (Student / Faculty / Admin)
* Student accounts generated via faculty-uploaded data
* Faculty accounts generated by admin
* Block/Unblock control for users
* Secure credential handling

---

## ⚙️ System Architecture

### 🔹 Core Principles

* **Single Source of Truth** (central database)
* **Real-Time Synchronization** across all roles
* **Strict Relational Data Model**
* **No duplicate or inconsistent data**

---

## 🗄️ Database Overview

Key entities:

* Users
* Credentials
* Achievements
* Verification Records
* Academic Records
* Attendance
* Subjects
* Departments / Courses / Sections
* Notifications

All modules are **interconnected via unique IDs** to ensure consistency.

---

## 🔄 Workflow

1. Faculty/Admin generates user credentials
2. Student logs into system
3. Student submits achievement with proof
4. Faculty verifies submission
5. Approved data is stored centrally
6. Academic records & attendance are added by faculty
7. Dashboards update in real-time
8. Reports and portfolios are generated dynamically
